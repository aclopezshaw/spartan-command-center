# ADR-0000: Decision title

- **Status:** Proposed
- **Date:** YYYY-MM-DD
- **Decision owners:** Name or role
- **Related SDCB tickets:** None or verified links
- **Supersedes:** None

## Context

Describe the current system, problem, constraints, and repository evidence. Distinguish implementation from intended behavior.

## Decision

State the proposed or accepted decision precisely.

## Consequences

### Positive

- Expected benefit.

### Negative

- Cost, limitation, or new responsibility.

### Risks

- Failure mode and mitigation.

## Alternatives considered

### Alternative name

Explain why it was not selected or what evidence could cause reconsideration.

## Implementation status

Use the documentation status vocabulary. Cite repository paths and relevant symbols. An Accepted decision is not automatically Implemented.

## Validation

Describe how contributors can prove that an implementation conforms to the decision.

## Reconsideration triggers

List scale, reliability, product, security, or cost conditions that should reopen the decision.
